# Steve Can Oshikko - 仕様書

## 概要

Minecraft Fabric Mod (Kotlin)。一定時間経過でプレイヤーに尿意が蓄積し、限界を超えると足元に「黄色い水」ブロックが生成される。おまるブロックを使用することで尿意を解消できる。

## 対象環境

- Minecraft: 1.21.4
- Fabric Loader: 0.18+
- Fabric Language Kotlin
- Kotlin: 2.3+

## ゲームメカニクス

### 尿意システム

| 項目 | 値 |
|------|-----|
| 尿意値の範囲 | 0 ~ 100 |
| 蓄積速度 | 1 / 10秒 (= 200 tick) |
| 満タンまでの時間 | 約16分40秒 (リアルタイム) |
| 限界値 (漏らす閾値) | 100 |
| おまる使用時のリセット値 | 0 |

- 尿意値はプレイヤーごとにサーバー側で管理する
- 死亡時は尿意値を 0 にリセットする
- 水を飲む (ポーション等) で蓄積速度が一時的に上がる、などの拡張は将来検討

### 尿意段階と演出

| 段階 | 尿意値 | HUD表示 | 効果 |
|------|--------|---------|------|
| 0: 正常 | 0 ~ 24 | 通常の顔 | なし |
| 1: 軽度 | 25 ~ 49 | やや青ざめた顔 | なし |
| 2: 中度 | 50 ~ 74 | 青い顔 | なし |
| 3: 重度 | 75 ~ 99 | 紫の顔、汗エフェクト | 移動速度 -20% (Slowness I) |
| 4: 限界突破 | 100 | 赤面 → 漏出アニメーション | 足元に黄色い水ブロック生成 |

### 漏出時の挙動

1. 尿意値が 100 に到達
2. プレイヤーの足元 (Y座標) に「黄色い水」ブロックを設置
3. プレイヤーに Slowness II を 5秒間付与 (ペナルティ)
4. 効果音を再生 (水の音)
5. 尿意値を 0 にリセット

## ブロック

### 黄色い水 (Pee Block)

- ID: `steveco:pee_block`
- 見た目: 黄色半透明のフルブロック (水っぽいテクスチャ)
- 光源レベル: 0
- 当たり判定: なし (通過可能)
- 60秒後に自然消滅する
- 設置時に周囲にパーティクルを表示

### おまる (Chamber Pot)

- ID: `steveco:chamber_pot`
- 見た目: 小さな壺型ブロック (カスタムモデル)
- 右クリックで使用 → 尿意値を 0 にリセット
- 使用時に効果音を再生
- クラフトレシピ: 鉄インゴット 5個

```
[   ] [   ] [   ]
[鉄 ] [   ] [鉄 ]
[鉄 ] [鉄 ] [鉄 ]
```

## HUD (ホットバー上の顔アイコン)

- 表示位置: ホットバーの左上 (空腹ゲージの隣あたり)
- サイズ: 16x16 ピクセル
- テクスチャ: 5段階のスティーブの顔スプライト (段階に応じて切り替え)
  - `textures/gui/face_normal.png` — 通常
  - `textures/gui/face_mild.png` — やや青ざめ
  - `textures/gui/face_moderate.png` — 青い顔
  - `textures/gui/face_severe.png` — 紫 + 汗
  - `textures/gui/face_critical.png` — 赤面
- クライアント側で描画 (サーバーから尿意値を同期)

## ネットワーク同期

- サーバー → クライアント: 尿意値の同期パケット (`S2CUrgencySync`)
  - 尿意値が変化するたびに送信 (10秒ごと)
- クライアントはローカルにキャッシュした尿意値でHUDを描画

## サウンド

| イベント | サウンドID |
|---------|-----------|
| 漏出時 | `steveco:pee_release` (水の流れるような音) |
| おまる使用時 | `steveco:chamber_pot_use` (短い水の音) |

## データの永続化

- 尿意値は Fabric Attachment API (`AttachmentRegistry`) で管理
- Identifier: `steveco:urgency` (Int)
- 内部的に NBT へ永続化される
